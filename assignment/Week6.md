# 1️⃣ 개념정리

## 01. 결과와 결론은 다르다

```
✅ 학습 목표 :
* 데이터 분석 결과와 해석을 통해 내리는 결론을 구분되어야 함을 이해한다.
```

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
계산과 분석을 해서 나온 결과물 = 결과 
그 결과가 목적에 대해 어떤 의미가 있는지 설명하는 것= 결론 
데이터 활용할 때 마지막으로 필요한건 결론이지만 대부분 결과만 말하는 문제점 有
!! 결론이란 결과에 기반해 도출해야 한다!! 
 - 데이터를 활용한다는 것은 판단과 행동으로 이어지는 결과물이 제시된 상태
 -  결과와 결론에 이르는 과정은 본질적으로 같음. (왜냐면 결론 = 결과 기반으로 도출되는 것)
 -  하지만 표현 방식은 다르다; 이 차이가 상대방에게 전달되는 방식이나 이해도에 영향 미침 
* 데이터 문해력에선 결과적으로 데이터를 분석해 획득한 정보를 목적에 맞게 활용하는 능력 필수적 
## 02. '결과'와 '결론'의 차이 : 데이터를 너무 단순하게 이해한 사례 

```
✅ 학습 목표 :
* 데이터 수치를 지나치게 단순하게 해석할 때 발생할 수 있는 오류를 사례를 통해 파악한다. 
* 동일한 결과라도 결론은 다양하게 나올 수 있음을 인식하고, 관점의 중요성을 설명할 수 있다. 
```

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
1) 결과와 결론을 혼동한 사례
특정 지역의 주택 가격·평균소득 데이터를 제시하고
"어디가 더 살기 좋은 곳인가?"라는 질문을 던지는 예시 등장.
많은 사람들이 단순히 숫자(평균 값)만 보고
“도시는 소득이 높고, 교외는 낮으니 도시가 더 낫다”고 결론 내림.
하지만 이는 평균값만 보고 성급히 결론을 내린 오류.
핵심: 데이터는 결과(수치)일 뿐,
바람직한 결론을 내리기 위해서는 맥락·가정·정책적 시사점을 고려해야 한다.

2) 표준편차와 분포를 통한 생각의 확장
단순 평균 비교가 아니라 분산·분포를 보고 해석해야 함
도시가 평균은 높지만 편차가 커
상위층·하위층 간 격차가 매우 클 수 있음
교외는 평균이 낮아도 상대적으로 안정적인 삶이 가능할 수 있음
즉 평균은 전체 진실이 아님 

3) 그래프를 통한 설득과 결론 도출
초콜릿 제품 판매량 데이터를 예시로 제시:
단순 수치만 보면 특정 지역이 우수해 보이지만
그래프를 통해 지역별 강점 품목이 다름을 파악할 수 있음.
이를 통해 제품별·지역별 맞춤 전략 수립 가능
(예: 특정 지역은 요금제가 잘 팔리니 그쪽 집중)

그래프는 단순 보여주기용이 아님
결론을 체계적으로 도출하는 도구

4) 좋은 질문이 결론을 이끈다
“어디가 더 좋은가?” 같은 모호한 질문은
데이터를 단순하게 소비하게 만듦
내가 어떤 삶을 원하는가?
어떤 정책 목표를 달성하고 싶은가
같은 질문이 필요

데이터는 목적에 맞는 질문이 있어야 의미가 생긴다


## 03. 결론을 낼 때 주의해야 할 점

```
✅ 학습 목표 :
* 결론 도출 시 반드시 점검해야할 부분을 습득한다. 
* 데이터를 통해 얻은 인사이트를 왜곡 없이 요약하는 방법을 설명할 수 있다. 
```

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->

1) 데이터만 보고 성급하게 결론 내리면 생기는 오류

“한국 사람은 빨리 먹는다, 걷는다, 말한다” 같은 문장들은
데이터 없이 선입견 강화한 예시

데이터를 봤을 때도 아마 그렇겠지? 식으로 자기 신념을 증명하려는 태도가 나타나기 쉬움 

핵심: 데이터가 말해준 것이냐, 내가 보고 싶은 것을 본 것이냐?를 스스로 점검해야 한다.

2) 데이터 해석에서 중요한 질문
내 생각은?이 아니라
데이터를 통해 뭘 알 수 있나?가 기준이 되어야 함

“한국인은 빨리 걷는다” 같은 결론은
근거 없이 문화적 편견을 강화하는 예시

3) 인지 편향의 역할
사람은 데이터를 수집할 때
이미 갖고 있는 믿음·선입견을 확인하려는 경향잇음 

특히 간단한 데이터만 보고
스스로 판단을 정당화하려는 경향이 강함

인간은 완전히 객관적일 수 없으므로
분석 과정에서 편향을 인식해야함 

4) 예시: 서울/도쿄/방콕/베이징 물가 비교
주어진 정보만 보고 어떤 도시가 비싸다고 단정할 수 없음 
교통비만 비싸거나, 식비만 비싸거나 등 부분적 정보로 전체 결론을 내릴 수 x 
결론: 하나의 지표로 전체 현실을 설명하려 하지 말 것.

5) 정답은 한 가지가 아니다
현실은 복잡하고 다면적이므로
    단순 결론을 찾으려 하면 오류 발생
데이터를 결론 도출 도구가 아니라
현실을 이해하는 다층적 창으로 써야함
